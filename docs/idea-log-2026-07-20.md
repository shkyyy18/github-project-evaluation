## 二、达标候选深度分析（Top 4）

### 1. JustVugg/colibri（90 分）—— 极致硬件约束下的工程奇观

**它做对了什么**
- 价值主张一句话击穿：「在 25GB 内存的消费级机器上跑 744B MoE，纯 C、零依赖」。数字对比（744B vs 25GB）自带传播性。
- 证据可验证：README 直接给出终端实录（`ready in 32s · resident 9.9 GB`）、Web dashboard 截图、把 19,456 个 expert 可视化成「活体皮层/3D 星系」的 Brain/Atlas 页面——把不可见的 MoE 路由变成可截图传播的视觉资产。
- 时机：踩中「大模型本地推理」与「超大 MoE 发布」双热点；issue #175 用「实测的 expert 主题亲和度」把社区互动变成内容生产。
- 分发：纯 C 零依赖 = 零安装摩擦，`./coli chat` 即 demo。

**能否复刻**：技术门槛极高（MoE 专家流式换入换出、int4 常驻内存管理），个人难以正面复刻。
**可复刻的差异化点**：不复刻引擎本身，复刻「可视化叙事」——为既有推理引擎（llama.cpp / ollama 生态）做 MoE 路由实时可视化插件/Dashboard，把「黑盒推理」变成可传播的视觉资产。这是 colibri 验证过、而它自己没空做的外围层。

### 2. langchain-ai/openwiki（86 分）—— agent 文档的"活文档"基础设施

**它做对了什么**
- 价值主张：CLI 为代码库写并**持续维护** agent wiki——关键词是「maintains」，一次性生成文档的工具很多，能跟着代码演进更新的很少。
- 分发教科书：npm 全局安装一步搞定；直接给 GitHub Actions / GitLab CI / Bitbucket Pipelines 三套现成 workflow 文件，把「CI 里自动提文档更新 PR」做成默认路径——文档工具变成了团队工作流的一环，这是留存设计。
- 双模式（code mode / personal mode）同时吃「代码库文档」和「个人知识脑」两个市场。
- 时机：AGENTS.md / agent 上下文工程成为共识的当口，「给 agent 看的文档」是新品类。

**能否复刻**：可以。核心是「代码库 ingest → LLM 综合 → wiki 落盘 → CI 增量更新」，没有深技术护城河，langchain 官方品牌才是。
**可复刻的差异化点**：做窄做深——只服务某一类仓库（如 Python 后端 monorepo）或只服务某一种产出（如面向 code review 的变更影响 wiki），并提供官方版没有的「文档可信度标注」（每段标注依据的源码行号与 commit，过时自动降级）。

### 3. elder-plinius/T3MP3ST（87 分）—— 多智能体自动化红队

**它做对了什么**
- 价值主张：autonomous red teaming platform，多 agent 进攻性安全元框架。作者 elder-plinius 本身就是越狱社区知名 IP，自带分发渠道。
- 证据密度高：README 15 处 demo/截图， AGPL-3.0 + 完整 topics，基础完整度满分。
- 时机：Agent 安全问题（提示注入、工具滥用）随 agent 普及而爆发，「用 agent 攻击 agent」是顺理成章的下一波。
- 社区：18 天 5,007 star、1,040 forks，fork/star 比 ~21% 为全场最高之一——安全社区有强烈的「拿来改」文化。

**能否复刻**：框架本身可以，但作者的个人 IP 与 jailbreak 社区信誉无法复刻；且 AGPL 限制了衍生商用。
**可复刻的差异化点**：不做进攻平台，做其镜像——**agent 防御侧评测**：把公开红队工具（含 T3MP3ST 这类）的 attack pattern 收集为 benchmark，给任意 agent 产品出「安全体检报告」。进攻工具越多，防御评测的需求越硬，且面向企业付费意愿强。

### 4. HUANGCHIHHUNGLeo/claude-real-video（83 分）—— 个人开发者的满分执行样本

**它做对了什么**
- 价值主张精准打击痛点：「粘贴 YouTube 链接给 ChatGPT，它读的是字幕不是画面；Claude 根本不吃视频文件」。场景感知抽帧（58 帧去重到 26 帧）+ whisper 转写 + 本地处理，一句话说清「为什么不是固定 1fps 采样」。
- 证据可验证性全场最强：60 秒真实 demo 视频、HN 首页徽章、PyPI 徽章、效果对比 GIF，README 55 处媒体证据。
- 分发组合拳：`pip install` + `npx skills add` 一条命令装进 Claude Code / Cursor / Codex / Copilot / Gemini CLI 等 50+ agent host——工具与 skill 双形态通吃 CLI 用户与 agent 用户。
- 商业化已跑通：免费版「看见」、Pro 版「理解」（镜头语言/情绪/声音事件），$19 创始价限时，个人项目闭环模板。
- 迭代密度高：adaptive 抽帧、字幕锚点、说话人分离（本地 45MB diarization 模型）——每个 edge case 都是一个功能点。

**能否复刻**：完全可以。技术栈是 ffmpeg 场景检测 + 帧去重 + whisper + 文件打包，无模型自研，一名工程师一周内可做出 MVP。
**可复刻的差异化点**：
1. 它输出「给人粘贴用」的文件夹，我们可以输出 **MCP server**，让 agent 直接按需调取帧/转写，省掉人工搬运；
2. 它靠 Claude/通用 VLM 读帧，我们可以加 **token 预算控制**（给定 token 上限自动决定帧数与分辨率）；
3. 它的 Pro 是人工设计的付费墙，开源复刻可直接覆盖「时间线理解」层并靠托管/批处理收费。

## 三、结论：最值得复刻的 1 个

**HUANGCHIHHUNGLeo/claude-real-video**

理由：
1. **需求已被验证且验证者是一个人**——个人开发者 20 天 1,751 star + HN 首页，说明痛点真实、市场尚未被巨头占领（grok-build/openwiki 那类官方项目无法复刻品牌，colibri 无法复刻技术）。
2. **复刻成本最低**——纯工程组装（ffmpeg + whisper + 打包），无模型训练、无硬件门槛、无数据壁垒，一周内可出 MVP。
3. **差异化路径清晰且它自己让出来了**——MCP 化（agent 原生调用而非人工粘贴）、token 预算自适应抽帧、把 Pro 的时间线理解层开源化，三点都建立在其 README 公开的能力边界上。
4. **退出/演化空间大**——「让 LLM 看视频」只是入口，同一管线可扩展到「让 LLM 看屏幕录像/会议/监控」，属于 agent 多模态输入的基础设施层。

风险提示：上游（Gemini 原生视频理解、各 agent host 内置视频输入）可能降维打击，窗口期估计 6-12 个月；差异化必须落在「本地处理 + agent 原生协议 + token 经济性」上，而不是再做一个更好的抽帧器。
